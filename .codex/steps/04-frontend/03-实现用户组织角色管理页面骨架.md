# Step 21：用户/组织/角色管理页面骨架（UI优先，功能逐步补齐）

## CODEX 执行指令（必须严格照做）
实现 3 个管理页面的 UI 骨架，并遵循 ui-ux-pro-max-skill：干净、克制、完整状态（loading/empty/error）。UTF-8（无 BOM）。

### 任务清单
1）页面：
- /users（用户列表）
- /org（组织树）
- /roles（角色与权限）
  2）每个页面必须包含：
- loading 状态
- empty 状态
- error 状态
  3）与后端联通（先读为主）：
- users：调用 GET /api/users
- org：调用 GET /api/org-units/tree
- roles：调用 GET /api/roles /api/permissions
  4）组件化：
- AppTable（列表）
- AppEmpty/AppLoading/AppError
- 基础表单弹窗占位（后续再接创建/修改）

### 交付物
- 3 个页面 + 基础组件
- 路由与侧边栏菜单

### 验收（你必须给出）
- 前端能打开三个页面
- 后端接口未就绪时能展示 error 状态（不白屏）
