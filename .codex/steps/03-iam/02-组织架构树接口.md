# Step 15：组织架构（树形）接口（创建/树查询/移动节点）

## CODEX 执行指令（必须严格照做）
实现 org_unit 树形管理的基础能力，重点包含“防止成环”。UTF-8（无 BOM）。

### 任务清单
1）建立 OrgUnitDO/Mapper/Service
2）DTO：
- OrgCreateReq（parentId、name、type）
- OrgMoveReq（id、newParentId）
- OrgTreeResp（树节点结构）
  3）接口：
- POST `/api/org-units` 创建节点
- GET `/api/org-units/tree` 查询整棵树
- PUT `/api/org-units/move` 移动节点
  4）必须实现“防环”：
- newParentId 不能是自己或自己的子孙节点
- path 字段维护（例如 /1/3/8/）

### 交付物
- OrgUnit DO/Mapper/Service/Controller
- 防环校验逻辑
- tree 构建逻辑（一次性查出所有节点再组装）

### 验收（你必须给出）
- curl：
    - 创建根节点、子节点
    - 查询 tree
    - 移动节点
- 证明防环：
    - 尝试把父节点移动到子节点下，返回 409（或 400）且有明确错误码（建议 ORG_5201）

### 禁止扩展
- 不做复杂组织权限继承（后续再做）
