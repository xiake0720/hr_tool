# Step 16：角色与权限基础管理（RBAC数据维护）

## CODEX 执行指令（必须严格照做）
实现角色、权限、绑定关系的基础接口。UTF-8（无 BOM）。

### 任务清单
1）建立 Role/Permission DO/Mapper/Service
2）DTO：
- RoleCreateReq（code、name）
- PermissionCreateReq（action、name）
- BindRolePermissionsReq（roleId、permissionIds）
- BindUserRolesReq（userId、roleIds）
  3）接口：
- POST `/api/roles`
- POST `/api/permissions`
- POST `/api/roles/{id}/permissions`
- POST `/api/users/{id}/roles`
- GET `/api/roles`（分页）
- GET `/api/permissions`（分页）

### 交付物
- Role/Permission 相关全套代码
- 绑定关系表操作（user_role、role_permission）

### 验收（你必须给出）
- curl：
    - 创建 role
    - 创建 permission
    - 绑定 permission 到 role
    - 绑定 role 到 user
- 确保唯一约束错误时返回 409（SYS_1002 或 IAM_5301）

### 禁止扩展
- 不做数据权限（只做功能权限）
