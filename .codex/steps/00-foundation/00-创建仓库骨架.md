# Step 00：创建仓库骨架（本地调试优先）

## CODEX 执行指令（必须严格照做）
你需要在仓库根目录执行以下任务，并确保所有新建/修改文件均为 **UTF-8（无 BOM）** 编码。

### 任务清单
1）确保创建下列目录（不存在则创建）：
- services/
- services/gateway-service/
- services/iam-service/
- web-frontend/
- infra/（仅放本地说明文档，本阶段不做 docker）
- steps/
- docs/（可选）

2）创建/覆盖根目录 `README.md`（中文）必须包含：
- 项目简介：HR工具平台 Phase 1（基础框架）
- 技术架构：Spring Cloud 微服务 + Nacos + OAuth2 Resource Server + MySQL + MyBatis + Redis + Kafka + Vue3
- 本地依赖地址（占位写法，不要写死密码）：
    - Nacos：http://localhost:8848/nacos
    - MySQL：localhost:3306，db=hr_iam_dev
    - Redis：localhost:6379
    - Kafka：localhost:9092
- 启动方式：
    - 后端：mvn 启动 gateway-service / iam-service
    - 前端：npm run dev
- Phase 1 范围：只做登录骨架、用户/组织/角色基础管理，不做简历解析等业务

3）创建/覆盖 `.gitignore`，至少包含：
- Java：target/、*.log、logs/
- Node：node_modules/、dist/
- IDE：.idea/、*.iml
- Env：.env、*.local、application-*.local.yml

4）在 `infra/` 下创建 `LOCAL_ENV.md`（中文），记录：
- 本地 Nacos/MySQL/Redis/Kafka 的启动方式（你不需要给出安装过程，只写“已安装后如何启动/检查”）
- 常见连通性检查命令示例（mysql/redis/kafka）

### 交付物
- README.md
- .gitignore
- infra/LOCAL_ENV.md
- 目录结构

### 你输出给我的结果格式
执行完成后输出：
1）创建/修改的文件清单  
2）下一步建议执行的 step 文件名（只写一个）

### 禁止扩展
- 不创建 docker-compose
- 不生成任何业务功能代码
